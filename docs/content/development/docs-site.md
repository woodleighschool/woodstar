# Docs Site

This site is built with [Docusaurus](https://docusaurus.io/) and lives under `docs/`. The pages you're reading are in `docs/content/`, kept apart from the working notes elsewhere in `docs/` so only the real documentation gets published.

## Layout

```text
docs/
  content/              the pages
  content/api/          generated API reference (see below)
  src/css/custom.css    theme overrides
  src/pages/index.tsx   landing page
  static/img/           logo and icons
  docusaurus.config.ts  site config
  sidebars.ts           autogenerated sidebar
```

The sidebar is autogenerated from the folder structure. Each top-level folder has a `_category_.json` for its label and order; pages set their own `sidebar_position`.

## Commands

Run these from `docs/`:

```bash
pnpm install
pnpm start        # local dev server with hot reload
pnpm build        # production build; also catches broken links and MDX errors
pnpm typecheck
```

`pnpm build` is the one that fails loudly on a broken link or a bad MDX file, so it's the check to trust before publishing.

## The API reference

The operation pages in the [API reference](../api/overview) are generated from `web/openapi.yaml`, the same spec the backend produces, using [docusaurus-openapi-docs](https://github.com/PaloAltoNetworks/docusaurus-openapi-docs). OpenAPI tag groups and tags define the integration and resource hierarchy in the sidebar.

Regenerate it after the spec changes:

```bash
pnpm gen-api-docs    # regenerate content/api from web/openapi.yaml
pnpm clean-api-docs  # remove the generated pages
```

The generated files under `content/api/` are output, not source. Don't edit them by hand; change the spec (or the backend that produces it) and regenerate. The hand-written part is the [overview](../api/overview) page that introduces the reference.

## Writing for this site

A few habits keep the docs readable:

- Write in plain present tense about how Woodstar works. No marketing, no roadmap.
- Keep the reference close to the thing: env vars, routes, commands, and processor names spelled out exactly.
- Preserve the domain words. Node key, label scope, pkginfo, Santa preflight, osquery distributed query: use them, don't paraphrase them away.
- Let the API reference own endpoint detail. Prose pages explain behaviour and link across rather than restating request shapes.
