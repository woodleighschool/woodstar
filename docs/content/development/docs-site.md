# Docs Site

The Docusaurus site lives under `docs/`. The authored documentation lives under `docs/content/` so existing project notes in `docs/` are not accidentally published.

## Layout

```text
docs/
  content/              authored docs
  src/css/custom.css    Docusaurus theme overrides
  src/pages/index.tsx   docs landing page
  static/img/           Woodstar logo and icons
  docusaurus.config.ts  site config
  sidebars.ts           sidebar generation
  package.json          docs toolchain
```

The site follows the direction of Qui's Docusaurus docs where it fits: local search, generated `llms.txt`, a minimal theme, autogenerated sidebars, and short pages that keep reference material close to the code.

## Commands

Run these from `docs/`:

```bash
pnpm install
pnpm typecheck
pnpm build
pnpm start
```

`pnpm build` catches broken links and MDX compile errors. `pnpm start` serves the docs locally for browser review.

## Writing Rules For This Site

- Do not write product marketing.
- Keep release claims out until releases exist.
- Preserve domain terms such as node key, label scope, pkginfo, Santa preflight, and osquery distributed query.
- Mark uncertainty by saying what the code shows and what still depends on operator choices.
- Prefer exact routes, env vars, package names, and commands over broad descriptions.
